# Phase 99 Empirical Model Intelligence Design

## Objective

Build local empirical model profiles from comparable, evidence-backed Forge runs. Catalog metadata and name-based role hints remain useful discovery aids, but they must be visibly labeled heuristic and must never be presented as measured performance.

## Authority Boundary

- The extension host owns ingestion, validation, cohort assignment, aggregation, persistence, and ranking.
- Models cannot write profile records, choose comparison cohorts, or promote heuristic labels to measured claims.
- Every sample retains its source artifact path, source digest, run identity, provenance, model identity, task identity, and comparison contract.
- Duplicate run/task/lane samples are rejected. Changed source bytes under an existing sample identity are treated as tampering.
- OpenRouter meta-routes such as `openrouter/auto` and `openrouter/pareto-code` are excluded because they do not identify one concrete model.

## Comparable Evidence

Profiles are partitioned into cohorts. A cohort key includes source kind, suite or common-contract digest, task input digest, judge or oracle digest, lane, assurance level, and tool-policy digest when available. Results from different cohorts may appear in one model summary, but head-to-head rankings are only valid inside one cohort.

Accepted sources in Phase 99:

1. Production benchmark reports with fixed suite integrity and per-task input/judge digests.
2. Branch-comparison reports whose report digest and candidate digests revalidate.
3. Scripted fixtures generated by the Phase 99 smoke test, retained as `scripted` provenance and never counted as live evidence.

Legacy weak-eval reports without stable task/judge digests remain unsupported instead of being silently pooled.

## Metrics

Each model/cohort profile reports sample count, live sample count, model-driven solve rate, Wilson 95% confidence interval, false-success rate, schema reliability, provider failures, cost per verified task, fallback dependence, and evidence references.

`measured` requires at least three live, verified samples in the same cohort. Smaller or scripted sets are explicitly `provisional`; zero accepted samples are `unmeasured`.

## Persistence

- `.forge/model-intelligence/latest.json`: current validated profile report.
- `.forge/model-intelligence/runs/<report-id>.json`: immutable report archive.
- `.forge/model-intelligence/latest-summary.md`: native-editor summary.

The report contains a canonical SHA-256 digest. Loading recomputes the digest, sample identities, cohort keys, profile metrics, and eligible rankings.

## Product Surface

- Commands rebuild, get, and open local model intelligence.
- Proof contains one collapsed `Model intelligence` control with rebuild/open actions and a compact measured/provisional summary.
- Model picker labels catalog/name scores as `heuristic`; empirical badges appear only when a validated host report supplies them.
- No provider call or credit spend is required to rebuild profiles.

## Honest Boundary

Phase 99 provides defensible local measurement and comparison mechanics. It does not create model quality data by itself. Until live comparable runs exist, Forge must show provisional scripted evidence or no measured evidence, not claim that one model is better.
