# Phase 93 Execution Contract And Assurance Implementation Plan

Lifecycle: `PLAN -> RECONCILE -> DOCUMENT -> IMPLEMENT -> VALIDATE -> REVIEW -> DOCUMENT CLOSE -> AAR`

## Bounded Objective

Implement the selected Phase 93 design without beginning background runners, customization imports, proof graphs, calibration, comparisons, empirical rankings or the Agent Gateway.

## Implementation Order

1. Add canonical contract types, compilation, normalization, digesting, revision comparison and assurance capability checks.
2. Persist active/history artifacts and normalize legacy sessions.
3. Gate `runStep`, steering, tool approval and terminal success on the active contract.
4. Add host commands and webview bridge for assurance selection, contract inspection and digest-bound confirmation.
5. Add the compact assurance chip and inline contract card.
6. Add focused/adversarial fixtures, extension-host coverage and visual states.
7. Run the complete release matrix, review against the design, close documents and record AAR.

## Acceptance Criteria

- No mutating/agentic provider work occurs before a pending contract is confirmed.
- The model cannot select assurance, alter a contract, confirm it or weaken its requirements.
- Standard is behavior-compatible with the current full harness.
- Verified success requires model-driven, fallback-free, independently reviewed, composite-green evidence.
- Audited is unavailable until isolation, calibration and attestation prerequisites are all proven.
- Contract widening creates a new revision and ask gate.
- One bottom composer remains; advanced details stay collapsed or native.

## Non-Goals

- No paid provider run.
- No remote service, background process or new mutation loop.
- No claim that Phase 97 calibration or Phase 96 attestation already exists.
- No Git operations.

## Planned Validation

`npm run test:execution-contract`, `npm test`, affected no-spend suites, `npm run test:visual`, `npm run test:e2e`, `npm run package`, package inspection, VS Code install and actual Antigravity smoke.

## Document Close

Pending implementation and executed evidence.
